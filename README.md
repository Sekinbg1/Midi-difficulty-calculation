# Midi-Difficulty-Calculation

MIDI 难度计算方法

1. 从三个方面计算难度
　※注意：以下的描述中：在同一时刻下，弹奏的所有音，即和弦 (组合音)/非和弦 (单音)，皆用“音”来表示；在和弦中的每一个音符，都用“音符”以表示，以此为区分；
　※先都放大 6 倍来计算，保证在计算单个音时，其“音数难度”和“跨度难度”始终为整数，不会导致后面的计算中，出现，由于除以 3，除不尽会造成小数；而且，这样计算下来，“十级曲目”对应的三种难度，就都刚好为 60 (保证了一定的精度)

　➢基础难度 (每两音之间的“时间间隔”)
　　●原理：音符之间的时间间隔越短，演奏难度越高
　　　※考虑到人的反应时间极限约为0.1s，即 100ms，故 100ms 对应“十级难度”
　　●公式：6000 ÷ 时间间隔 (毫秒)
　　●基准：需要将 100ms 间隔对应“十级难度” (值为 60)，所以用 6000 来除 (6 倍放大)

　➢音符数量难度
　　●原理：同时演奏的音符越多（和弦越复杂），难度越高
　　　※1 个音符最简单；5 个音符为最难，对应“十级难度”
　　●基准：1 个音符难度为 2，5 个音符难度为 10 (原始难度)
　　　　　　1 个音符难度为 12，5 个音符难度为 60 (6 倍放大)

　➢跨度难度 (音程跨度)
　　●原理：和弦中从根音到冠音的跨度越大，手部伸展要求越高，难度越大
　　　※12 个半音 (一个八度) 为最难，对应“十级难度”
　　●基准：1 个半音跨度难度为 5/6，12 个半音 (一个八度) 跨度难度为 10
　　　　　　1 个半音跨度难度为 5，12 个半音 (一个八度) 跨度难度为 60 (6 倍放大)

2. 单个音难度计算
　　对于每个音，综合三个方面计算难度：
　※由于，非和弦 (单音) 不存在跨度，故其“跨度难度”恒为 0
　●和弦 (组合音) 难度公式：(基础难度＋音数难度＋跨度难度) ÷ 3
　●非和弦 (单音) 难度公式：(基础难度＋音数难度) ÷ 2

3. 通道难度计算
　●遍历通道中的所有音 (从第二个音开始)
　●计算相邻音之间的时间间隔对应的难度
　●对所有有效转换的难度取平均值
　●最后除以6得出原始难度，即对应 0~10 级难度值的范畴

4. 特殊处理：钢琴多音轨分离
　单独计算钢琴部分的难度：

　➢钢琴识别
　　●将 GM 标准中 0~15 号乐器 (钢琴类和键盘类) 识别为钢琴

　➢多音轨处理策略
　　●钢琴类：如果同一通道在多个轨道中有数据，分别计算每个轨道的难度，然后取平均值
　　●非钢琴类：将同一通道在所有轨道中的数据合并后统一计算难度
　这种设计考虑到了钢琴曲目常将左右手分配到不同音轨的情况。

5. 整体难度计算流程
5.1. 解析 MIDI 文件：读取文件头、轨道数据，提取音符、和弦、BPM 等信息
5.2. 识别有效通道：只处理包含实际音符数据的通道 (避免空通道影响平均值)
5.3. 按通道计算难度：
　●钢琴通道：多音轨分别处理
　●其他乐器：合并处理
5.4. 计算总体难度：对所有有效通道的难度取平均值

6. 难度等级划分
　最终结果按以下标准分级：

　●专业级：8~10 分
　●高级：6~8 分
　●中级：4~6 分
　●初级：2~4 分
　●入门级：0~2 分

➢技术特点
6.1. 时间精度：使用毫秒级时间计算，基于 MIDI 文件的 tick 和 BPM 转换
6.2. 和弦识别：将相同起始时间的音符合并为和弦处理
6.3. 内存安全：包含边界检查和异常处理
6.4. 兼容性：支持 MIDI 格式 0、1、2，处理 SMPTE 时间格式
6.5. 用户友好：提供详细的解析过程输出和乐器名称显示

　　方法综合考虑了演奏的技术要求 (速度、复杂度、手部跨度)，对于钢琴部分单独进行了处理，能够相对准确地评估 MIDI 的演奏难度。
