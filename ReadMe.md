# Midi-Difficulty-Calculation
MIDI难度计算
　通过三个方面来计算MIDI难度，并遵循项目规范的各项原则。

一、综合三个方面计算难度
　※注意：以下的描述中：在同一时刻下，弹奏的所有音，即和弦 (多个音符)/单音 (单个音符)，皆用“音”来表示；在和弦中的每一个音符，都用“音符”以表示，以此为区分。
　※先都放大6倍来计算，保证在计算单个音时，其“音数难度”和“跨度难度”始终为整数，不会导致后面的计算中，出现，由于除以3，除不尽会造成小数；而且，这样计算下来，“十级曲目”对应的三种难度，就都刚好为60 (保证了一定的精度)。

　➢基础难度
　　●原理：基于每两音之间的“时间间隔”计算，时间间隔越短 (演奏速度越快)，难度越大
　　　※考虑到人的极限反应时间约为0.1s，遂将100ms对应“十级难度”
　　●公式：6000÷时间间隔 (ms)
　　●基准：需要将100ms对应“十级难度”，所以用6000来除 (6倍放大)

　➢音数难度
　　●原理：同时演奏的音符越多 (和弦越复杂)，难度越大
　　　※1个音符为最简单；5个音符为最难，对应“十级难度”
　　●公式：音数×12
　　●基准：1个音符难度为2，5个音符难度为10 (原始难度)
　　　　　　1个音符难度为12，5个音符难度为60 (6倍放大)

　➢跨度难度 (音程跨度)
　　●原理：和弦中从“根音”到“冠音”的跨度越大，即手部伸展要求越大，难度越大
　　　※12个半音 (一个八度) 为最难，对应“十级难度”
　　●公式：跨度×5 (跨度＝冠音编号－根音编号)
　　●基准：1个半音难度为5/6，12个半音 (一个八度) 难度为10 (原始难度)
　　　　　　1个半音难度为5，12个半音 (一个八度) 难度为60 (6倍放大)

二、单个音的难度计算规则
　对于每个音，综合三个方面计算难度：
　※由于，单音不存在跨度，故其“跨度难度”恒为0

　对于“首音”和“后续音”分别计算：
　➢首音
　　●单音：音数难度
　　●和弦：(音数难度＋跨度难度)÷2

　➢后续音
　　●单音：(基础难度＋音数难度)÷2
　　●和弦：(基础难度＋音数难度＋跨度难度)÷3

三、聚合策略
　➢通道内聚合
　　●将通道内所有音的难度累加后除以实际音的数量
　　●关键还原步骤：最终结果再除以6，来还原放大系数，即对应0~10级难度值的范畴

　➢多音轨处理
　　●如果单通道包含多个有效音轨，则该通道难度取各音轨难度的平均值

　➢总体难度
　　●所有有效通道难度的平均值 (总难度还需除以有效通道数)
　　●仅包含至少一个音符或和弦的通道被视为“有效”

四、算法特点
　●6倍放大策略：所有中间计算都使用6倍放大，最后统一还原
　●有效数据判定：严格过滤空通道，避免无效数据干扰
　●时间基准：以100ms间隔对应“十级难度”(60) 为基准
　●钢琴特殊处理：识别钢琴类乐器 (程序编号0~15)，但计算逻辑与其他乐器相同

　➢项目规范的核心原则
　　●首音单独处理
　　●有效数据判定
　　●通道间采用平均值而非累加
　　●放大系数的正确还原
　　●严格区分单音与和弦

五、整体难度计算流程
　➢解析MIDI文件：读取文件头、轨道数据，提取音符、和弦、BPM等信息
　➢识别有效通道：只处理包含实际音符数据的通道 (避免空通道影响平均值)
　➢计算通道难度：对所有“有效音轨”的难度取平均值
　➢计算总体难度：对所有“有效通道”的难度取平均值

六、难度等级划分
　最终结果按以下标准分级：
　　●专业级：8~10分
　　●高级：6~8分
　　●中级：4~6分
　　●初级：2~4分
　　●入门级：0~2分

　➢技术特点
　　●时间精度：使用毫秒级时间计算，基于MIDI文件的tick和BPM转换
　　●和弦识别：将相同起始时间的音符合并为和弦处理
　　●内存安全：包含边界检查和异常处理
　　●兼容性：支持MIDI格式0、1、2，处理SMPTE时间格式
　　●用户友好：提供详细的解析过程输出和乐器名称显示

　此方法综合考虑了演奏的技术要求 (速度、复杂度、手部跨度)，能够相对准确地评估MIDI的演奏难度
